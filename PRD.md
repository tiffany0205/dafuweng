# 跳棋抽奖活动 - 需求文档 (PRD)

## 1. 活动概述

### 1.1 活动名称
**幸运跳棋 · Lucky Jump**

### 1.2 活动简介
用户通过完成日常任务（签到、充值、邀请好友等）获取**抽奖机会**，消耗抽奖机会在 36 格棋盘上掷骰子前进。棋盘上分布着各种奖励和陷阱，每走完一圈排名前进。活动结束后，根据排行榜排名发放实物和虚拟奖品。

### 1.3 核心玩法
```
完成任务 → 获得抽奖机会 → 掷骰子走棋盘 → 触发格子事件 → 积累圈数/格子 → 排行榜竞争 → 赢取奖品
```

---

## 2. 用户流程

### 2.1 新用户流程
```
访问活动页 → 注册账号 → 登录 → 活动首页
    → 查看任务列表 → 完成任务获取抽奖机会
    → 进入棋盘 → 消耗机会掷骰子
    → 查看排行榜 → 活动结束领取奖品
```

### 2.2 老用户流程
```
登录 → 活动首页 → 签到/充值/邀请 → 获得机会 → 走棋盘 → 冲排名
```

## 3. 账号系统

### 3.1 注册
- 用户需要注册账号才能参与活动
- 注册信息：**用户名 + 密码 + 邀请码（选填）**
- 注册成功后系统自动生成唯一邀请码
- 若使用邀请码注册，邀请人与被邀请人建立绑定关系

### 3.2 登录
- 使用用户名 + 密码登录
- 登录后进入活动首页

## 4. 任务系统

用户完成任务可获得抽奖机会，任务列表展示所有任务及完成状态。

### 4.1 每日签到（连续签到）

| 项目 | 说明 |
|------|------|
| 规则 | 每天可签到1次 |
| 普通奖励 | 签到即得 **5次** 抽奖机会 |
| 连续签到奖励 | 连续签到每满7天（第7天、第14天、第21天…），当天得 **10次** 抽奖机会 |
| 断签惩罚 | 中途断签，连续天数归零，重新从第1天开始计算 |
| 限制 | 每日0点重置签到状态，不可补签 |
| 状态 | 今日未签到 / 今日已签到 + 当前连续签到天数 |

**示例：**
```
第1天签到 → 5次    第7天签到 → 10次   第8天签到 → 5次
第2天签到 → 5次     ...               ...
第3天签到 → 5次    第13天签到 → 5次   第14天签到 → 10次
...
```
> 断签后重新签到，从第1天重新算起，得5次。

### 4.2 充值任务
| 项目 | 说明 |
|------|------|
| 规则 | 每充值 **10 USDT** 获得 **10次** 抽奖机会 |
| 奖励 | 按充值金额计算，取10 USDT的整数倍 |
| 示例 | 充10 USDT = 10次机会，充25 USDT = 20次机会，充100 USDT = 100次机会 |
| 特点 | 可重复完成，无上限 |
| 状态 | 进度条：已充X USDT，可获Y次机会 |

### 4.3 邀请好友
| 项目 | 说明 |
|------|------|
| 规则 | 好友通过你的邀请码注册成功 |
| 奖励 | 每成功邀请1位好友 = **5次** 抽奖机会 |
| 特点 | 可重复完成，无上限 |
| 状态 | 已成功邀请 X 人 |

### 4.4 好友首次充值奖励
| 项目 | 说明 |
|------|------|
| 规则 | 你邀请的好友**首次充值达标**（≥10 USDT） |
| 奖励 | 邀请人获得 **10次** 抽奖机会 |
| 限制 | 每位好友仅首次达标充值赠送，后续该好友再充值不再赠送 |
| 特点 | 被动触发，好友首充达标自动到账 |
| 状态 | 已获得 X 次好友充值奖励 |

## 5. 棋盘设计（36格）

### 5.1 棋盘概述
- 棋盘共 **36个格子**，编号 1-36
- 第1格为**起点**
- 走完第36格后回到第1格，算完成**1圈**
- 棋盘为环形设计，类似大富翁/飞行棋

### 5.2 格子分布

| 格子编号 | 类型 | 触发效果 |
|----------|------|----------|
| 1 | 起点 | 起点，无特殊效果 |
| 4 | 奖励 | +0.01 USDT |
| 6 | 奖励 | +50 PHP |
| 8 | 奖励 | +10000 VND |
| 10 | 冰冻 | 冰冻！需消耗1次抽奖机会解冻 |
| 13 | 电池 | 额外前进2格 |
| 16 | VIP升级 | VIP等级+1 |
| 19 | 炸弹 | 回到起点（第1格），圈数不变 |
| 22 | 奖励 | +0.01 USDT |
| 25 | 冰冻 | 冰冻！需消耗1次抽奖机会解冻 |
| 28 | 奖励 | +50 PHP |
| 31 | 奖励 | +10000 VND |
| 34 | 电池 | 额外前进2格 |
| 36 | VIP升级 | VIP等级+1 |

> 未列出的格子为**普通格**，到达后无特殊效果。

### 5.3 特殊效果详解

**冰冻**
- 用户进入冰冻状态，**无法掷骰子**
- 需要消耗 **1次抽奖机会** 解冻
- 解冻后留在原格子，恢复正常移动

**炸弹**
- 棋子立即**回到起点（第1格）**
- 圈数保持不变
- 已走过的格子数保留在累计记录中

**电池**
- 自动额外前进 **2格**
- 如果触发电池后又落在特殊格，继续触发效果（连锁）

**VIP升级**
- 用户VIP等级 +1
- 纯荣誉性奖励，不影响棋盘移动

**奖励格**（0.01 USDT / 50 PHP / 10000 VND）
- 到达后即时获得对应奖励
- 奖励记录在用户账户中

## 6. 抽奖机会

### 6.1 获得途径

| 来源 | 说明 | 数量 |
|------|------|------|
| 每日签到 | 每天签到1次 | 5次（连续7天倍数得10次）|
| 充值 | 每充值10 USDT | 10次/每10U |
| 邀请好友 | 好友通过邀请码注册 | 5次/人 |
| 好友首充 | 被邀请好友首次充值≥10U | 10次（仅首次）|

### 6.2 消耗途径

| 用途 | 说明 |
|------|------|
| 掷骰子 | 消耗1次机会，掷骰子(1-6随机)走棋盘 |
| 解冻 | 冰冻状态下消耗1次机会解冻 |

### 6.3 机会列表
用户可查看所有抽奖机会的**获得记录**，包括：
- 来源（签到/充值/邀请/好友充值）
- 获得时间
- 状态（已使用/未使用）

## 7. 排行榜

### 7.1 排名规则
排行榜实时更新，按以下优先级排序：

| 优先级 | 排序依据 | 说明 |
|--------|----------|------|
| 第一优先 | 完成圈数 | 圈数越多排名越高 |
| 第二优先 | 当前格子数 | 相同圈数下，格子数越多排名越高 |
| 第三优先 | 到达时间 | 相同圈数和格子数下，越早到达排名越高 |

### 7.2 排行展示
- 展示 **前20名** 用户
- 每行显示：**排名**、用户名、圈数、格子位置、VIP等级
- **前10名旁边标注对应奖品**，一眼看到冲到这个名次能拿什么
- 用户自己的排名始终可见（即使不在前20）

### 7.3 排名与奖品对照（排行榜上展示）

| 排名 | 奖品 | 排行榜展示 |
|------|------|------------|
| 🥇 第1名 | iPhone 16 Pro | 排名旁显示 🎁 iPhone 16 Pro |
| 🥈 第2名 | 500 USDT | 排名旁显示 🎁 500 USDT |
| 🥉 第3名 | 400 USDT | 排名旁显示 🎁 400 USDT |
| 第4名 | 300 USDT | 排名旁显示 🎁 300 USDT |
| 第5名 | 200 USDT | 排名旁显示 🎁 200 USDT |
| 第6-10名 | 100 USDT | 排名旁显示 🎁 100 USDT |
| 第11-20名 | 无 | 仅展示排名，无奖品标识 |

> **设计要点：** 排行榜上每个名次旁边直接展示该名次对应的奖品，用户一眼就能看到「我冲到第几名能拿什么」，增强竞争动力。前10名的奖品标识建议用金色/高亮样式突出，11-20名无奖品形成对比，刺激用户追赶。

## 9. 页面结构

### 9.1 登录页
- 用户名 + 密码登录
- 跳转注册页入口
- 登录成功后进入活动首页

### 9.2 注册页
- 用户名 + 密码 + 邀请码（选填）
- 注册成功自动登录

### 9.3 活动首页（核心页面）
页面从上到下包含以下模块：

**a) 用户信息栏**
- 头像、用户名、VIP等级
- 当前排名、已完成圈数、当前格子

**b) 抽奖机会余额**
- 剩余可用次数（醒目展示）
- 点击进入机会明细列表

**c) 任务列表**
- 签到任务（签到按钮 + 今日状态）
- 充值任务（进度条 + 已获机会数）
- 邀请任务（分享邀请码 + 已邀请人数）
- 好友充值奖励（已获得机会数）
- 每个任务显示当前状态

**d) 快捷入口**
- 「进入棋盘」按钮 → 跳转棋盘游戏页
- 「排行榜」按钮 → 跳转排行榜页

### 9.4 棋盘游戏页
- **棋盘可视化**：36格环形/蛇形排列，高亮当前所在格
- **骰子区**：点击掷骰子按钮，骰子动画（1-6随机）
- **状态栏**：当前格/圈数/剩余机会/是否冰冻
- **移动动画**：棋子从当前格移动到目标格
- **事件弹窗**：到达特殊格时弹出效果说明
- **冰冻状态**：掷骰子按钮变灰，显示「解冻」按钮

### 9.5 排行榜页
- **前20名列表**：排名、头像、用户名、圈数、格子数、VIP等级
- **我的排名卡片**：始终固定在底部或列表中高亮
- **奖品展示区**：展示各名次对应奖品

### 9.6 中奖名单页（活动结束后）
- 展示最终获奖用户列表
- 用户查看自己的中奖状态

## 12. 非功能需求

- 适配手机端（移动优先设计）
